# Admin Rules

Record admin-only behaviors and review constraints.

## System Code Admin

- `페이지 관리`는 페이지 메타데이터와 기본 VIEW 권한의 시작점이다.
- 페이지 등록 시 `PAGE_CODE_VIEW` 기본 기능이 자동 생성되어야 한다.
- 자동 생성된 VIEW 기능은 권한 그룹에 자동 부여하지 않는다. 권한 할당은 운영자가 수동 검토한다.
- 페이지 수정 시 자동 생성된 기본 VIEW 기능 메타데이터만 동기화한다. 수동 액션 기능(`CREATE`, `UPDATE`, `DELETE`)은 자동 동기화 대상이 아니다.
- 페이지 삭제 시 기본 VIEW 외의 액션 기능이 남아 있으면 삭제를 차단한다.
- 페이지 삭제가 허용되면 기본 VIEW 기능, 권한 그룹 매핑, 사용자 예외권한을 함께 정리해야 한다.

## Menu Management Orchestration

- `메뉴 관리`는 기존 화면을 덮어쓰는 화면이 아니라 `메뉴 -> 페이지 -> 기본 VIEW 기능` 자동 등록의 허브로 확장한다.
- 정상 동작 중인 기존 화면은 유지하고, 더 이상 메뉴에 노출하지 않을 경우 `삭제` 대신 `useAt = N` 숨김 처리한다.
- 메뉴 관리에서 새 8자리 페이지 메뉴를 만들 때는 `COMTCCMMNDETAILCODE`, `COMTNMENUINFO`, 기본 `PAGE_CODE_VIEW` 기능, 초기 정렬 순서가 함께 생성되어야 한다.
- 메뉴 관리에서 생성된 새 페이지는 `UI_PAGE_MANIFEST`, `UI_COMPONENT_REGISTRY`, `UI_PAGE_COMPONENT_MAP` draft registry도 함께 생성하고, screen command에서 이벤트/API/스키마 후보를 자동으로 보여줘야 한다.
- 새 페이지 코드는 선택한 6자리 그룹 메뉴 하위에서 다음 suffix를 계산해 충돌 없이 자동 생성한다.
- 페이지 위치 조정은 메뉴 순서 저장으로 처리하고, 코드 체계를 바꾸는 식으로 정렬을 맞추지 않는다.
- 사이트맵 노출 여부는 메뉴 노출 여부와 분리해 관리한다. 사이트맵 전용 노출 제어는 별도 `사이트맵 관리` 메뉴에서 다루는 것을 기본 원칙으로 둔다.

## Authority Chain

- 관리자 권한은 항상 `메뉴 -> 페이지 -> 기능 -> 권한 그룹 -> 회원/부서 할당 -> 사용자 예외권한 -> 감사로그` 순서로 이어져야 한다.
- 권한 편집 화면은 현재 로그인한 관리자가 실제로 부여 가능한 페이지/기능만 노출해야 한다.
- 권한 저장 시 현재 관리자가 수정할 수 없는 기능은 제거하지 말고 기존 상태를 보존해야 한다.
- 권한 그룹 생성, 권한 그룹 기능 저장, 부서 권한 매핑, 회원 권한 저장, 관리자 권한 저장은 모두 감사로그를 남겨야 한다.
- `MASTER`가 아닌 경우에는 항상 `instt_id` 또는 회사 ID 범위 안에서만 권한 조회/부여/수정이 가능해야 한다.
- `instt_id` 범위 제한은 화면 숨김으로 끝내지 말고 컨트롤러, 서비스, 매퍼/SQL까지 같은 조건으로 강제해야 한다.

## System Menu Scope

- `/admin/system/*` 화면은 마스터 관리자만 접근할 수 있어야 한다.
- `/admin/api/admin/system/*` 기능 API도 마스터 관리자만 접근할 수 있어야 한다.
- 예외적으로 `/admin/system/access_history`와 같은 시스템 로그 메뉴는 마스터 관리자와 시스템 관리자까지 허용할 수 있다.
  이 경우 시스템 관리자는 반드시 자기 `instt_id` 범위 로그만 조회할 수 있어야 한다.
- `/admin/member/login_history`를 포함한 모니터링 메뉴도 마스터 관리자만 접근할 수 있어야 한다.
- `/admin/content/*` 화면과 `/admin/api/admin/content/*` 기능 API도 마스터 관리자만 접근할 수 있어야 한다.
- `/admin/external/*` 화면과 `/admin/api/admin/external/*` 기능 API도 마스터 관리자만 접근할 수 있어야 한다.
- 시스템 메뉴의 좌측 메뉴 노출과 사이트맵 노출도 마스터 관리자에게만 허용한다.
- 시스템/모니터링/콘텐츠/외부연계 메뉴 기능 권한이 별도로 등록되어 있더라도, 마스터 관리자가 아니면 화면 진입과 기능 실행을 모두 차단한다.

## Admin Restoration Track

- 현재 관리자 화면 복원 우선순위는 `auth_group`, `auth_change`, `dept_role_mapping`, `member_edit`, `admin_account` 5개 페이지다.
- 위 5개 페이지는 원본 관리자 템플릿 구조를 먼저 복원하고, 이후 React 마이그레이션 화면 또는 확장 기능을 다시 얹는다.
- 복원 전에 넣는 변경은 원본 템플릿 복원 후에도 유지되는 백엔드 계약, 권한 범위 제한, 감사로그, 데이터 모델 정리 위주로 제한한다.

## Review Evidence

- 페이지 등록 후 `COMTCCMMNDETAILCODE`, `COMTNMENUINFO`, `COMTNMENUFUNCTIONINFO`에 기대한 코드가 생성되는지 확인한다.
- 페이지 삭제 차단 시 남은 기능 코드와 기능 관리 화면 링크가 함께 노출되는지 확인한다.
- 삭제 전 영향도 숫자와 삭제 후 정리 완료 메시지가 일치하는지 확인한다.
- 한글/영문 템플릿이 같은 동작을 유지하는지 확인한다.

## Irreversible Actions

- 페이지 코드는 메뉴 코드 및 기본 VIEW 기능 prefix로 사용되므로 운영상 불변 키로 취급한다.
- 페이지 삭제는 메뉴 메타데이터만 지우는 작업이 아니라 권한 참조 정리까지 포함하므로 영향도 확인 없이 수행하면 안 된다.
